# BotNexus.AgentCore

Stateful agent with tool execution, event streaming, and steering messages. Built on `BotNexus.Providers.Core` for unified LLM interactions.

## Installation

Add BotNexus.AgentCore to your C# project:

```bash
dotnet add package BotNexus.AgentCore
```

Or include in your `.csproj`:

```xml
<ItemGroup>
    <PackageReference Include="BotNexus.AgentCore" Version="1.0.0" />
</ItemGroup>
```

## Quick Start

```csharp
using BotNexus.AgentCore;
using BotNexus.AgentCore.Configuration;
using BotNexus.AgentCore.Types;
using BotNexus.Providers.Core;

// Create agent configuration
var options = new AgentOptions(
    InitialState: new AgentInitialState(
        SystemPrompt: "You are a helpful assistant.",
        Model: new LlmModel("claude-3-5-sonnet", "anthropic"),
        Tools: [],
        Messages: []
    ),
    Model: new LlmModel("claude-3-5-sonnet", "anthropic"),
    LlmClient: llmClient,
    ConvertToLlm: async (messages, ct) =>
    {
        // Convert AgentMessage timeline to provider Message[] for the LLM
        return await MessageConverter.ConvertAsync(messages, ct);
    },
    TransformContext: async (messages, ct) =>
    {
        // Optional: filter, summarize, or rewrite messages
        return messages;
    },
    GetApiKey: async (provider, ct) =>
    {
        // Resolve API key for the provider
        return Environment.GetEnvironmentVariable($"{provider.ToUpper()}_API_KEY");
    },
    GetSteeringMessages: null,
    GetFollowUpMessages: null,
    ToolExecutionMode: ToolExecutionMode.Sequential,
    BeforeToolCall: null,
    AfterToolCall: null,
    GenerationSettings: new SimpleStreamOptions
    {
        Temperature = 0.7,
        MaxTokens = 2048
    },
    SteeringMode: QueueMode.All,
    FollowUpMode: QueueMode.All
);

// Create agent instance
var agent = new Agent(options);

// Subscribe to events
using var subscription = agent.Subscribe(async (evt, ct) =>
{
    switch (evt)
    {
        case MessageUpdateEvent update:
            Console.Write(update.ContentDelta);
            break;
        case ToolExecutionEndEvent toolEnd:
            Console.WriteLine($"Tool {toolEnd.ToolName} completed: {toolEnd.IsError}");
            break;
    }
    await Task.CompletedTask;
});

// Run a prompt
var result = await agent.PromptAsync("What is 2+2?");
var lastMessage = result[^1] as AssistantAgentMessage;
Console.WriteLine($"Response: {lastMessage?.Content}");
```

## Core Concepts

### AgentMessage vs LLM Message

The agent maintains an internal timeline of **AgentMessages** that represent the conversation history:
- **UserMessage**: Text input from the user (with optional images for multimodal models).
- **AssistantAgentMessage**: Responses generated by the LLM (includes tool calls and token usage).
- **ToolResultAgentMessage**: Results from executed tools.
- **SystemAgentMessage**: System-level prompts (usually prepended to context).

These AgentMessages are **converted to provider Messages** at the LLM call boundary via the `ConvertToLlmDelegate`. This separation allows:
- Storing UI-only metadata in the timeline without confusing the LLM.
- Filtering, summarizing, or rewriting messages before they reach the provider.
- Supporting custom message types by extending the AgentMessage hierarchy.

### Message Flow

```
User Input (PromptAsync)
    ↓
Add to Timeline
    ↓
Drain Steering Messages (mid-run corrections)
    ↓
Convert to Provider Messages (ConvertToLlm)
    ↓
Transform Context (filter/summarize)
    ↓
Call LLM Provider
    ↓
Receive Assistant Response
    ↓
Execute Tools (if requested)
    ↓
Emit Tool Results
    ↓
Check for More Turns (steering messages queued?)
    ↓
Repeat or Finalize
    ↓
Drain Follow-Up Messages (ContinueAsync or new PromptAsync)
    ↓
Return All New Messages
```

## Event Flow

### PromptAsync() Event Sequence

A single `PromptAsync("hello")` call emits the following event sequence:

```
1. AgentStartEvent
2. TurnStartEvent
3. MessageStartEvent (for the user prompt)
4. MessageEndEvent (for the user prompt)
5. TurnStartEvent (second turn for LLM call)
6. MessageStartEvent (assistant generation starts)
7. MessageUpdateEvent* (zero or more streaming chunks)
8. MessageEndEvent (assistant generation complete)
9. [If tools called] ToolExecutionStartEvent* (one per tool)
10. [If tools called] ToolExecutionEndEvent* (one per tool)
11. TurnEndEvent
12. [If follow-ups queued] repeat from step 2
13. AgentEndEvent (final event, agent becomes idle)
```

**Notes:**
- All events are awaited in subscription order before the run proceeds.
- Listeners receive the active `CancellationToken` for the run.
- Tool execution can be **Sequential** or **Parallel** (configured by `ToolExecutionMode`).

### With Tool Calls

When the assistant requests tool execution:

```
MessageEndEvent (assistant message with tool calls)
    ↓
[For each tool, in parallel if ToolExecutionMode.Parallel]
  ToolExecutionStartEvent (arguments validated)
    ↓
  BeforeToolCallDelegate (hook for validation/blocking)
    ↓
  ExecuteAsync (tool implementation)
    ↓
  AfterToolCallDelegate (hook for result transformation)
    ↓
  ToolExecutionEndEvent (result finalized)
    ↓
  ToolResultAgentMessage added to timeline
```

### Event Types Reference

| Event Type | Emitted When | Key Fields |
| --- | --- | --- |
| **AgentStartEvent** | Run starts | `Timestamp` |
| **AgentEndEvent** | Run completes or aborts | `Messages` (full timeline), `Timestamp` |
| **TurnStartEvent** | LLM call begins | `Timestamp` |
| **TurnEndEvent** | Turn completes (message + tool results) | `Message`, `ToolResults`, `Timestamp` |
| **MessageStartEvent** | Assistant generation starts | `Message`, `Timestamp` |
| **MessageUpdateEvent** | Streaming chunk received | `Message`, `ContentDelta`, `ToolCallId`, `ArgumentsDelta`, `Timestamp` |
| **MessageEndEvent** | Assistant generation complete | `Message`, `Timestamp` |
| **ToolExecutionStartEvent** | Tool execution begins (after validation) | `ToolCallId`, `ToolName`, `Args`, `Timestamp` |
| **ToolExecutionUpdateEvent** | Partial tool result (reserved for future) | `ToolCallId`, `PartialResult`, `Timestamp` |
| **ToolExecutionEndEvent** | Tool execution completes | `ToolCallId`, `ToolName`, `Result`, `IsError`, `Timestamp` |

## Agent Options

Create an `AgentOptions` record to configure the agent:

```csharp
public record AgentOptions(
    AgentInitialState? InitialState,                    // Initial state seed
    LlmModel Model,                                     // Model to use
    ConvertToLlmDelegate ConvertToLlm,                 // Message conversion (required)
    TransformContextDelegate TransformContext,          // Context transformation (required)
    GetApiKeyDelegate GetApiKey,                        // API key resolution (required)
    GetMessagesDelegate? GetSteeringMessages,           // Steering message producer
    GetMessagesDelegate? GetFollowUpMessages,           // Follow-up message producer
    ToolExecutionMode ToolExecutionMode,                // Sequential or Parallel
    BeforeToolCallDelegate? BeforeToolCall,             // Pre-execution hook
    AfterToolCallDelegate? AfterToolCall,               // Post-execution hook
    SimpleStreamOptions GenerationSettings,             // Temperature, maxTokens, etc.
    QueueMode SteeringMode,                             // All or OneAtATime
    QueueMode FollowUpMode,                             // All or OneAtATime
    string? SessionId = null);                          // Optional session identifier
```

### Key Options

- **InitialState**: Seeds the agent with system prompt, model, tools, and messages. Copies are made at construction time — changes to InitialState after construction have no effect.
- **Model**: The LLM model to use (e.g., `new LlmModel("claude-3-5-sonnet", "anthropic")`).
- **ConvertToLlm**: Must convert `IReadOnlyList<AgentMessage>` to provider `Message[]`. Called before each LLM invocation.
- **TransformContext**: Optional message filtering/summarization. Called after ConvertToLlm.
- **GetApiKey**: Resolves API keys on demand for the provider. Return `null` if key unavailable.
- **ToolExecutionMode**: Controls whether tools run sequentially or in parallel:
  - `Sequential`: Tools execute one after another.
  - `Parallel`: Tools execute concurrently (ensure thread safety).
- **SteeringMode / FollowUpMode**: Controls queue consumption:
  - `All`: Drain all queued messages at once.
  - `OneAtATime`: Consume one message per iteration.

## Agent State

The `Agent.State` property exposes mutable runtime state:

```csharp
public class AgentState
{
    // Configuration (can be mutated mid-run)
    public string? SystemPrompt { get; set; }
    public LlmModel Model { get; set; }
    public ThinkingLevel? ThinkingLevel { get; set; }
    public IReadOnlyList<IAgentTool> Tools { get; set; }
    public IReadOnlyList<AgentMessage> Messages { get; set; }

    // Runtime state (read-only, managed internally)
    public bool IsStreaming { get; }
    public AssistantAgentMessage? StreamingMessage { get; }
    public IReadOnlySet<string> PendingToolCalls { get; }
    public string? ErrorMessage { get; }
}
```

### Common Patterns

```csharp
// Add a tool
agent.State.Tools = [..agent.State.Tools, myNewTool];

// Update system prompt
agent.State.SystemPrompt = "New system instruction";

// Modify model
agent.State.Model = new LlmModel("gpt-4", "openai");

// Set thinking level (for models that support extended reasoning)
agent.State.ThinkingLevel = new ThinkingLevel { Budget = 1024, Type = "simple" };

// Check if streaming is active
if (agent.State.IsStreaming)
{
    var partialMessage = agent.State.StreamingMessage;
}

// Check for errors
if (agent.State.ErrorMessage is not null)
{
    Console.WriteLine($"Error: {agent.State.ErrorMessage}");
}
```

### Thread Safety

- All state mutations are visible immediately.
- Mutations during in-flight runs do **not** affect the current run (internal snapshots are taken).
- `Tools` and `Messages` properties copy the provided collection to prevent external mutation.

## Methods

### Prompting

#### `PromptAsync(string, CancellationToken)`

Start a new run with a text message:

```csharp
var result = await agent.PromptAsync("What's the weather?");
```

#### `PromptAsync(AgentMessage, CancellationToken)`

Start a run with a single message:

```csharp
var message = new UserMessage("Tell me a joke");
var result = await agent.PromptAsync(message);
```

#### `PromptAsync(IReadOnlyList<AgentMessage>, CancellationToken)`

Start a run with multiple messages (e.g., system prompt + user input):

```csharp
var messages = new AgentMessage[]
{
    new SystemAgentMessage("You are a math tutor."),
    new UserMessage("What is 7 * 8?")
};
var result = await agent.PromptAsync(messages);
```

**Throws:** `InvalidOperationException` if the agent is already running.

### State Management

#### `State` Property

Access mutable runtime state:

```csharp
agent.State.SystemPrompt = "New prompt";
agent.State.Tools = myTools;
agent.State.Messages = []; // Clear timeline
```

#### `Reset()`

Clear all state and message queues:

```csharp
agent.Reset();
// Clears: Messages, Tools, error state, streaming state, all queues
```

### Control

#### `ContinueAsync(CancellationToken)`

Resume execution without adding a new message. Useful for retries or multi-turn workflows:

```csharp
// After PromptAsync completes, if there are queued steering messages:
await agent.ContinueAsync();
```

**Throws:** `InvalidOperationException` if the last message is from the assistant.

#### `AbortAsync()`

Cancel the currently running agent loop:

```csharp
await agent.AbortAsync();
// Sets status to Aborting, cancels internal CancellationTokenSource
// Waits for active run to settle
```

#### `WaitForIdleAsync(CancellationToken)`

Wait for the agent to become idle:

```csharp
await agent.WaitForIdleAsync();
// Returns immediately if already idle
```

#### `Status` Property

Check current execution status:

```csharp
if (agent.Status == AgentStatus.Running)
{
    Console.WriteLine("Agent is busy");
}
// Returns: Idle, Running, or Aborting
```

### Events

#### `Subscribe(Func<AgentEvent, CancellationToken, Task>)`

Register an event listener:

```csharp
using var sub = agent.Subscribe(async (evt, ct) =>
{
    if (evt is MessageUpdateEvent update)
    {
        Console.Write(update.ContentDelta);
    }
    await Task.CompletedTask;
});
```

**Returns:** An `IDisposable` subscription handle. Unsubscribe by disposing.

**Notes:**
- Listeners are awaited in subscription order before the run proceeds.
- Each listener receives the active `CancellationToken` for the run.
- Multiple listeners can be subscribed simultaneously.
- New listeners registered during a run receive events from the next turn boundary onward.

## Steering and Follow-up

### Steering Messages

Inject messages mid-run to correct or steer the agent:

```csharp
// While a run is active, enqueue a steering message
agent.Steer(new UserMessage("Wait, I meant to ask about the Moon, not the Sun."));
```

Steering messages are drained before the next LLM call (at the next turn boundary). Use `SteeringMode` to control consumption:
- **All** (default): All queued steering messages are injected together.
- **OneAtATime**: One steering message per LLM call.

### Follow-up Messages

Automatically continue the conversation after a run completes:

```csharp
// Enqueue a follow-up message
agent.FollowUp(new UserMessage("Can you expand on that?"));

// After the current run settles, follow-up triggers a new PromptAsync automatically
```

Follow-up messages are drained after `PromptAsync` completes (and no tool calls are pending). Use `FollowUpMode` to control consumption.

### Queue Management

```csharp
// Clear queues manually
agent.ClearSteeringQueue();
agent.ClearFollowUpQueue();
agent.ClearAllQueues();
```

## Custom Message Types

Extend `AgentMessage` to support custom metadata or UI-specific information:

```csharp
public sealed record CustomUIMessage(string Content, string UIHint) 
    : AgentMessage("custom");

// Add to timeline
agent.State.Messages = [..agent.State.Messages, new CustomUIMessage("Status: busy", "spinner")];

// In ConvertToLlm, filter out custom messages so they don't reach the LLM
public async Task<IReadOnlyList<Message>> ConvertToLlm(
    IReadOnlyList<AgentMessage> messages,
    CancellationToken ct)
{
    var filtered = messages
        .Where(m => m is not CustomUIMessage)
        .ToList();
    return await provider.ConvertAsync(filtered, ct);
}
```

## Tools

### Defining Tools

Implement `IAgentTool` to create a tool:

```csharp
public class SearchTool : IAgentTool
{
    public string Name => "search";
    public string Label => "Web Search";

    public Tool Definition => new Tool
    {
        Name = "search",
        Description = "Search the web for information",
        InputSchema = new JsonSchema
        {
            Type = "object",
            Properties = new()
            {
                ["query"] = new() { Type = "string", Description = "Search query" }
            },
            Required = ["query"]
        }
    };

    public async Task<IReadOnlyDictionary<string, object?>> PrepareArgumentsAsync(
        IReadOnlyDictionary<string, object?> arguments,
        CancellationToken cancellationToken = default)
    {
        // Validate and coerce arguments
        var query = arguments["query"]?.ToString();
        if (string.IsNullOrWhiteSpace(query))
        {
            throw new ArgumentException("Query cannot be empty");
        }

        return new Dictionary<string, object?> { ["query"] = query.Trim() };
    }

    public async Task<AgentToolResult> ExecuteAsync(
        string toolCallId,
        IReadOnlyDictionary<string, object?> arguments,
        CancellationToken cancellationToken = default,
        AgentToolUpdateCallback? onUpdate = null)
    {
        var query = arguments["query"]?.ToString();
        var results = await PerformSearch(query, cancellationToken);
        
        return new AgentToolResult(
            [new AgentToolContent(AgentToolContentType.Text, results)]
        );
    }

    private async Task<string> PerformSearch(string query, CancellationToken ct)
    {
        // Implementation
        return $"Results for '{query}'...";
    }
}

// Register the tool
agent.State.Tools = [new SearchTool()];
```

### Error Handling

Tool errors are caught and converted to error tool results:

```csharp
public async Task<AgentToolResult> ExecuteAsync(
    IReadOnlyDictionary<string, object?> arguments,
    CancellationToken cancellationToken = default)
{
    try
    {
        var result = await FetchData(arguments, cancellationToken);
        return new AgentToolResult([new AgentToolContent(AgentToolContentType.Text, result)]);
    }
    catch (TimeoutException ex)
    {
        // Throw to convert to error result
        throw new InvalidOperationException($"Fetch timeout: {ex.Message}");
    }
}
```

When an exception is thrown:
- It is caught by the agent loop.
- A `ToolResultAgentMessage` is created with `IsError = true`.
- The error message is included in the result.
- A `ToolExecutionEndEvent` is emitted with `IsError = true`.
- The LLM receives the error and can decide how to proceed.

### Hooks

#### Before Tool Call (`BeforeToolCallDelegate`)

Validate, log, or block tool calls before execution:

```csharp
var options = new AgentOptions(
    // ... other options ...
    BeforeToolCall: async (context, ct) =>
    {
        // Access context.ToolName, context.Args, context.Message
        if (context.ToolName == "dangerous_tool" && !IsAuthorized())
        {
            return new BeforeToolCallResult(Block: true, BlockReason: "Unauthorized");
        }
        return null; // Allow execution
    }
);
```

#### After Tool Call (`AfterToolCallDelegate`)

Transform or override tool results:

```csharp
AfterToolCall: async (context, ct) =>
{
    // Access context.ToolName, context.Args, context.Result, context.IsError
    if (context.ToolName == "price_lookup" && context.Result.Content.Count > 0)
    {
        // Transform result before LLM sees it
        var transformed = FilterSensitiveData(context.Result);
        return new AfterToolCallResult(Content: transformed.Content);
    }
    return null; // Use original result
}
```

## Low-Level API (AgentLoopRunner)

For advanced use cases, interact directly with the agent loop:

```csharp
using BotNexus.AgentCore.Loop;

// Build configuration
var loopConfig = new AgentLoopConfig(
    Model: agent.State.Model,
    ConvertToLlm: options.ConvertToLlm,
    TransformContext: options.TransformContext,
    GetApiKey: options.GetApiKey,
    GetSteeringMessages: options.GetSteeringMessages,
    GetFollowUpMessages: options.GetFollowUpMessages,
    ToolExecutionMode: options.ToolExecutionMode,
    BeforeToolCall: options.BeforeToolCall,
    AfterToolCall: options.AfterToolCall,
    GenerationSettings: options.GenerationSettings
);

// Build context
var context = new AgentContext(
    SystemPrompt: agent.State.SystemPrompt,
    Messages: agent.State.Messages,
    Tools: agent.State.Tools
);

// Run the loop
var newMessages = await AgentLoopRunner.RunAsync(
    prompts: [new UserMessage("hello")],
    context: context,
    config: loopConfig,
    emit: async (evt) => Console.WriteLine(evt),
    cancellationToken: CancellationToken.None
);

// Or continue without new prompts
var continuedMessages = await AgentLoopRunner.ContinueAsync(
    context: context,
    config: loopConfig,
    emit: async (evt) => Console.WriteLine(evt),
    cancellationToken: CancellationToken.None
);
```

**Notes:**
- `AgentLoopRunner` is a static API with no state.
- Useful for custom agent implementations or integrations.
- The high-level `Agent` class wraps this for concurrency control, subscription management, and queue handling.

## Thread Safety

- **Agent**: Thread-safe. Multiple threads can call `PromptAsync`, `Steer`, `FollowUp`, `Subscribe`, and query `Status` simultaneously.
- **AgentState**: Thread-safe property accessors. Mutations are visible immediately but do not affect in-flight runs.
- **AgentLoopRunner**: Stateless. Safe to call from any thread.
- **Tools**: If `ToolExecutionMode.Parallel` is used, tool implementations must be thread-safe.

## Differences from pi-mono

| Feature | pi-mono (TypeScript) | BotNexus.AgentCore (C#) |
| --- | --- | --- |
| Abort signal | `AbortSignal` | `CancellationToken` |
| Unsubscribe | Return function | `IDisposable` subscription |
| Streaming | `AsyncIterable` | `IAsyncEnumerable` / `LlmStream` |
| Thinking off | Explicit `"off"` value | `ThinkingLevel? = null` |
| Concurrency | Single-threaded | Multi-threaded (`SemaphoreSlim`, locks) |
| Data model | Mutable classes | Records (immutable) + mutable `AgentState` |
| Provider reference | `@mariozechner/pi-ai` | `BotNexus.Providers.Core` |
| Event stream | `Channel<AgentEvent>` | `Func<AgentEvent, Task>` callbacks |
