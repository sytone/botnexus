    Your First AI Agent¶

    Build and chat with a custom BotNexus agent in about 15 minutes.

    By the end of this tutorial, you will:

    • run BotNexus locally
    • configure one AI provider
    • create your first agent
    • chat with it in the Web UI
    • customize its behavior with a system prompt

    Tip

    This is a hands-on tutorial. Copy and run each step in order.

    What you'll build¶

    You will create an agent named My First Agent that:

    • uses OpenAI gpt-4.1
    • responds with a friendly, concise style
    • runs in the BotNexus Web UI

    1) Prerequisites¶

    Make sure you have:

    1. .NET 10 SDK
    2. git
    3. One API key from a supported provider (OpenAI or Anthropic)

    Check your tools:

    dotnet --version
git --version

    You should see version output for both commands.

    Warning

    Keep API keys private. Never commit them to source control.

    2) Clone and build BotNexus¶

    Clone the repository:

    git clone https://github.com/sytone/botnexus.git
cd botnexus

    Build the solution:

    dotnet build BotNexus.slnx

    You should see output ending with something like:

    Build succeeded.

    If build fails, resolve that first before continuing.

    3) Configure a provider¶

    BotNexus reads configuration from ~/.botnexus/config.json.

    • Windows: C:\Users\<your-user>\.botnexus\config.json
    • macOS/Linux: ~/.botnexus/config.json

    Create or edit that file and add this minimal OpenAI provider config:

    {
  "version": 1,
  "providers": {
    "openai": {
      "apiKey": "your-api-key-here",
      "defaultModel": "gpt-4.1"
    }
  }
}

    Tip

    Replace "your-api-key-here" with your real OpenAI API key.

    4) Create your first agent¶

    In the same ~/.botnexus/config.json, add an agents section alongside the providers:

    {
  "version": 1,
  "providers": {
    "openai": {
      "apiKey": "your-api-key-here",
      "defaultModel": "gpt-4.1"
    }
  },
  "agents": {
    "my-first-agent": {
      "displayName": "My First Agent",
      "provider": "openai",
      "model": "gpt-4.1",
      "enabled": true
    }
  }
}

    That's a complete minimal configuration. BotNexus will load the agent on startup.

    Tip

    See the Configuration Reference for all available agent options including tools, sub-agents, and isolation strategies.

    5) Add a system prompt¶

    Create a system prompt file for your agent. By default, BotNexus looks for prompt files in the agent's directory:

    ~/.botnexus/agents/my-first-agent/system-prompt.md

    Create that file with your agent's personality:

    You are a helpful AI assistant named Botty.
You are friendly, concise, and always try to help.
When asked a question, give a clear and direct answer.

    Then reference it in your config by adding systemPromptFile:

    "my-first-agent": {
  "displayName": "My First Agent",
  "provider": "openai",
  "model": "gpt-4.1",
  "systemPromptFile": "system-prompt.md",
  "enabled": true
}

    6) Run the gateway¶

    From the repository root, start the API gateway:

    dotnet run --project src/gateway/BotNexus.Gateway.Api

    When startup completes, open:

    http://localhost:5005

    You should see the BotNexus Web UI load in your browser.

    7) Chat with your agent¶

    In the Web UI:

    1. Select My First Agent (or my-first-agent)
    2. Send a message like:
    Hi Botty! Give me 3 tips for learning BotNexus quickly.

    You should see a friendly, concise response generated by your agent.

    If you get an auth/provider error, recheck:

    • provider name (openai) matches a key in providers
    • model name (gpt-4.1) is valid for your provider
    • API key is correct

    8) Customize behavior¶

    Now change your agent's personality by editing the system prompt file.

    For example, update ~/.botnexus/agents/my-first-agent/system-prompt.md:

    You are Botty, a practical mentor.
Respond with numbered steps and short explanations.
Keep answers under 100 words when possible.

    Then:

    1. Stop the running gateway (Ctrl+C)
    2. Restart it with the same command:
    dotnet run --project src/gateway/BotNexus.Gateway.Api

    Send the same chat message again.

    You should notice a different response style that matches your new prompt.

    Tip

    System prompts are the fastest way to shape agent behavior without writing code.

    As a next step, you can add tools to your agent so it can do more than just chat.

    You now have a working BotNexus agent and a repeatable workflow for creating more.
