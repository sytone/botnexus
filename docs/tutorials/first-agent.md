# Your First AI Agent

Build and chat with a custom BotNexus agent in about 15 minutes.

By the end of this tutorial, you will:

- run BotNexus locally
- configure one AI provider
- create your first agent
- chat with it in the Web UI
- customize its behavior with a system prompt

!!! tip
    This is a **hands-on tutorial**. Copy and run each step in order.

## What you'll build

You will create an agent named **My First Agent** that:

- uses OpenAI `gpt-4.1`
- responds with a friendly, concise style
- runs in the BotNexus Web UI

---

## 1) Prerequisites

Make sure you have:

1. **.NET 10 SDK**
2. **git**
3. **One API key** from a supported provider (OpenAI or Anthropic)

Check your tools:

```bash
dotnet --version
git --version
```

You should see version output for both commands.

!!! warning
    Keep API keys private. Never commit them to source control.

---

## 2) Clone and build BotNexus

Clone the repository:

```bash
git clone https://github.com/sytone/botnexus.git
cd botnexus
```

Build the solution:

```bash
dotnet build BotNexus.slnx
```

You should see output ending with something like:

```text
Build succeeded.
```

If build fails, resolve that first before continuing.

---

## 3) Configure a provider

BotNexus reads configuration from `~/.botnexus/config.json`.

- Windows: `C:\Users\<your-user>\.botnexus\config.json`
- macOS/Linux: `~/.botnexus/config.json`

Create or edit that file and add this minimal OpenAI provider config:

```json
{
  "Providers": {
    "openai": {
      "Type": "OpenAI",
      "ApiKey": "your-api-key-here",
      "Models": {
        "gpt-4.1": {
          "DeploymentName": "gpt-4.1"
        }
      }
    }
  }
}
```

!!! tip
    Replace `"your-api-key-here"` with your real key.

---

## 4) Create your first agent

In the **same** `~/.botnexus/config.json`, add an `Agents` section:

```json
{
  "Agents": {
    "my-first-agent": {
      "DisplayName": "My First Agent",
      "Provider": "openai",
      "Model": "gpt-4.1",
      "SystemPrompt": "You are a helpful AI assistant named Botty. You are friendly, concise, and always try to help.",
      "Channels": ["WebUI"]
    }
  }
}
```

If your file is new, it can look like this combined example:

```json
{
  "Providers": {
    "openai": {
      "Type": "OpenAI",
      "ApiKey": "your-api-key-here",
      "Models": {
        "gpt-4.1": {
          "DeploymentName": "gpt-4.1"
        }
      }
    }
  },
  "Agents": {
    "my-first-agent": {
      "DisplayName": "My First Agent",
      "Provider": "openai",
      "Model": "gpt-4.1",
      "SystemPrompt": "You are a helpful AI assistant named Botty. You are friendly, concise, and always try to help.",
      "Channels": ["WebUI"]
    }
  }
}
```

---

## 5) Run the gateway

From the repository root, start the API gateway:

```bash
dotnet run --project src/gateway/BotNexus.Gateway.Api
```

When startup completes, open:

```text
http://localhost:5005
```

You should see the BotNexus Web UI load in your browser.

---

## 6) Chat with your agent

In the Web UI:

1. Select **My First Agent** (or `my-first-agent`)
2. Send a message like:

```text
Hi Botty! Give me 3 tips for learning BotNexus quickly.
```

You should see a friendly, concise response generated by your agent.

If you get an auth/provider error, recheck:

- provider name (`openai`)
- model name (`gpt-4.1`)
- API key value

---

## 7) Customize behavior

Now change your agent personality by editing `SystemPrompt`.

For example, update:

```json
"SystemPrompt": "You are Botty, a practical mentor. Respond with numbered steps and short explanations."
```

Then:

1. stop the running gateway (`Ctrl+C`)
2. restart it with the same command:

```bash
dotnet run --project src/gateway/BotNexus.Gateway.Api
```

Send the same chat message again.

You should notice a different response style that matches your new prompt.

!!! tip
    System prompts are the fastest way to shape agent behavior without writing code.

As a next step, you can add tools to your agent so it can do more than chat.
